# API Contracts: [Feature Name]

> This directory contains API/interface contracts for the feature.
> Generated by `/sdd-plan`. Each contract file defines a specific interface.
>
> **Convention:** One file per resource or interface area.
> - REST: `[resource].md` e.g., `users.md`, `orders.md`
> - Events/WebSockets: `[event-group].md` e.g., `notifications.md`
> - gRPC/GraphQL: `[service].md`

---

## Example Contract File Structure

Each contract file in this directory should follow this format:

```markdown
# Contract: [Resource Name]

## POST /[resource]

**Purpose:** [What this endpoint does]

### Request
\`\`\`json
{
  "field": "type — description"
}
\`\`\`

### Response 201
\`\`\`json
{
  "id": "string",
  "field": "value"
}
\`\`\`

### Errors
| Status | Code | When |
|---|---|---|
| 400 | VALIDATION_ERROR | Required field missing |
| 409 | CONFLICT | [Specific conflict condition] |
```
